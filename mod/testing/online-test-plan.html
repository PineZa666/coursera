Online A/B Test Plan — Add Activity Flow

Component & Manipulation

  • Original (A): Standard form with title, category, minutes, and a submit button.
  • Redesign (B): Adds 25/50/75 minute presets, reorders fields, and enlarges the submit CTA.

Assignment

  • Users visit distinct URLs: add-activity-a.html vs add-activity-b.html.

Primary Metric (Binary)

  • Conversion: submitted an activity (yes/no).

Secondary Metrics

  • Time to submit (milliseconds).
  • Preset used (yes/no, minutes value).

Instrumentation

  • Local logging via logABEvent in app.js records view, preset, and submit with timestamps.

Analysis

  • Compute counts of submitted vs not submitted for A and B.
  • Apply Chi‑squared test to compare conversion rates.
  • Interpretations: - If B > A significantly: presets and CTA improve engagement. - If no difference: redesign may be neutral; reconsider microcopy or placement. - If A > B: presets caused confusion; simplify or educate.

Risks & Mitigations

  • Small sample sizes: supplement with in‑person tests and longer collection.
  • Repeat visitors: deduplicate by session heuristics.
